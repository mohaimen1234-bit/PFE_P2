3. Docker Commands  
Check Docker is running

Safe -ish cleanup:

More aggressive cleanup:

Careful: --volumes can remove unused database volumes.

13. Common Problems and Fixes  
Problem: Docker storage full  
Fix: 
docker system prune -a 
Then:  
wsl --shutdown  
Restart Docker Desktop.  
 
Problem: Minikube API server broken  
Fix: 
minikube delete  
wsl --shutdown  
minikube start --driver=docker --kubernetes -version=v1.32.0  
 
Problem: service not found  
You used wrong service names earlier.  
Correct services:  
backend -service  
frontend -service  
postgres -service  Not:  
cmms -backend  
cmms -frontend  
 
Problem: frontend login calls wrong URL  
Bad:  
localhost:8081 inside frontend pod  
Good inside Kubernetes:  
http://backend -service:8081  
In next.config.mjs, backend proxy should point to:  
http://backend -service:8081  
 
Problem: browser login gives 403  
Fix was backend CORS/security:  
allowedOriginPatterns:  
http://127.0.0.1:*  
http://localhost:*  
http://192.168.*.*:*  
Also:  
/api/auth/** permitAll  
 
Problem: backend pod CreateContainerConfigError  
Cause:  
BOOTSTRAP_ADMIN_PASSWORD missing in postgres -secret  
Fix: 
kubectl apply -f k8s/secrets.yaml  
kubectl rollout restart deployment/cmms -backend -n cmms   
Problem: Argo CD app Progressing because of ingress  
Fix: 
minikube addons enable ingress  
kubectl get pods -n ingress -nginx  
kubectl get ingress -n cmms  
 
Problem: Helm not recognized  
Install Helm manually, then:  
helm version
